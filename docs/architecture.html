服務一覽

6 個 .NET 服務 + 1 個前端

外加 SQL Server 與 RabbitMQ 兩個基礎設施容器，共 9 個容器。各服務只透過 HTTP 與事件溝通。

MatchService
核心 API + 資料庫。唯一寫 MatchDb 者；REST API；Dapper 呼叫 Stored Procedure；啟動時套用 DB 腳本。
IngestionJob
賽前擷取。每 5 分鐘抓運彩 Pre feed，合併中英、換算賠率，寫入賽程與盤口。
LiveIngestion
即時擷取。每 ~12 秒抓 Live feed，解析比分/狀態/分鐘與 in-play 賠率；並每輪直接發比賽時鐘。
OutboxPublisher
事件發布。把資料庫 Outbox 裡的事件可靠地發布到 RabbitMQ（at-least-once）。
RealtimeService
即時推送。訂閱事件，經 SignalR Hub 推給所有前端連線。
NotificationService
通知。訂閱比分/狀態/賠率事件並記錄（個人化通知的基礎）。
frontend
React 即時看板。賽事看板、玩法明細、賠率走勢圖；以 nginx 提供。
Contracts
共用契約（函式庫，非服務）。傳輸 DTO 與事件型別，被各服務參考以保證型別一致。
